# Утилиты (utils) для проекта deb_scripts

В этой директории находятся различные утилиты, используемые в проекте `deb_scripts`.

## Структура

- `logger.py` - модуль для настройки и использования логгера с эмоджи
- `subprocess_utils.py` - модуль для безопасного выполнения системных команд
- `update_utils.py` - модуль для обновления операционной системы
- `permission_utils.py` - модуль для проверки прав доступа (root)
- `args_utils.py` - модуль для парсинга аргументов командной строки

## logger.py

Модуль предоставляет настроенный логгер с эмоджи для улучшения UX.

### Функции:

- `get_logger(name)` - возвращает экземпляр логгера с заданным именем

### Уровни логирования:

- `INFO` - для основных этапов выполнения
- `DEBUG` - для детального логирования
- `ERROR` - для сообщений об ошибках

## subprocess_utils.py

Модуль для безопасного выполнения системных команд через subprocess.

### Функции:

- `run(command, **kwargs)` - выполнение команды с настроенными параметрами безопасности

### Особенности:

- Автоматически обрабатывает ошибки
- Проверяет код возврата команды
- Позволяет передавать дополнительные параметры в subprocess

## update_utils.py

Модуль для обновления операционной системы через apt.

### Функции:

- `update_os()` - выполняет обновление списков пакетов и установку обновлений

### Алгоритм работы:

1. Выполняет `apt update` для обновления списков пакетов
2. Выполняет `apt upgrade` для установки обновлений
3. Логирует процесс и возможные ошибки

## permission_utils.py

Модуль для проверки прав доступа (требуется root).

### Функции:

- `check_root()` - проверяет, запущен ли скрипт с правами root

### Поведение:

- Выбрасывает `PermissionError`, если скрипт запущен не от имени root
- Возвращает `None`, если права root присутствуют

## args_utils.py

Модуль для парсинга аргументов командной строки.

### Функции:

- `parse_args()` - парсит аргументы командной строки и возвращает namespace

### Поддерживаемые аргументы:

- `--mode` - режим работы (dev/prod)
  - `dev` - включает DEBUG-логирование и пропускает некоторые проверки
  - `prod` - стандартный режим работы (по умолчанию)
