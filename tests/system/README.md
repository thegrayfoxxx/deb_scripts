# Системные тесты

Каталог `tests/system/` содержит тесты, которые работают не только с Python-кодом, но и с реальными системными командами, пакетным менеджером, `sysctl`, сетевыми утилитами и системным окружением контейнера.

## Когда использовать

Системные тесты нужны для проверки:

- доступности системных команд (`apt`, `curl`, `sysctl`, `iptables`, `ufw`, `docker`, `systemctl`)
- поведения сервисов в реальном Linux-окружении
- совместимости с привилегированной контейнерной средой
- реального взаимодействия с файловой системой, сетью и системными настройками

## Состав

- `test_system_commands.py` — базовые проверки системного окружения
- `test_*_service_system.py` — smoke-сценарии для отдельных сервисов

## Запуск

Рекомендуемый способ:

```bash
docker compose build test
docker compose run --rm test pytest -m system -q -rs
```

Запуск одного файла:

```bash
docker compose run --rm test pytest tests/system/test_docker_service_system.py -q -rs
```

Только медленные системные тесты:

```bash
docker compose run --rm test pytest -m "system and slow" -q -rs
```

## Ограничения среды

Даже внутри контейнера часть сценариев зависит от особенностей ядра и доступных системных сервисов. Поэтому:

- не все модули ядра будут доступны
- `systemd` и `systemctl` могут работать ограниченно
- `sysctl` может быть недоступен или урезан
- сетевые smoke-проверки могут быть пропущены при неподходящем окружении
- отдельные тесты корректно используют `skip`, если среда не соответствует минимальным требованиям

Текущие ожидаемые причины `skip`:

- отсутствие `sysctl` для части BBR-сценариев
- неподходящее сетевое окружение для отдельных command-smoke тестов
- отсутствие нужных привилегий для части UFW-сценариев

## Правила для новых системных тестов

- используйте маркер `@pytest.mark.system`
- не полагайтесь на состояние хоста
- учитывайте, что контейнер одноразовый и запускается в `privileged`-режиме
- проверяйте реальные эффекты только там, где это оправдано
- если поведение зависит от среды, явно используйте `pytest.skip(...)` с понятной причиной
- для побочных операций очищайте состояние в рамках теста или делайте сценарий идемпотентным
