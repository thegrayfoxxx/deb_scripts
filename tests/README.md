# Запуск тестов

Тесты предназначены для запуска в Docker-контейнере:

Тесты предназначены для запуска в Docker-контейнере:

```bash
# Сборка контейнер
docker compose build

# Запуск всех тестов
docker compose run --rm test

# Запуск только интеграционных тестов
docker compose run --rm test -m integration

# Запуск только юнит-тестов
docker compose run --rm test -m unit

# Запуск только системных тестов
docker compose run --rm test pytest -m system -v

# Запуск медленных системных тестов
docker compose run --rm test pytest -m "system and slow" -v

# Для интерактивной отладки внутри контейнера
docker compose run --rm test /bin/bash

# Для получения подробного отчета о покрытии
docker compose run --rm test --cov-report=html
```

## Особенности тестирования

- Все тесты используют `pytest` и `unittest.mock`
- Юнит-тесты: проверяют отдельные методы и классы изолированно
- Интеграционные тесты: проверяют взаимодействие между компонентами (сервис ↔ утилиты, интерфейс ↔ сервис)
- Используются mock-объекты для изоляции тестируемых компонентов
- Тесты проверяют как успешные сценарии, так и случаи ошибок
- Учитываются особенности системных команд и прав доступа

## Типы тестов

### Юнит-тесты
- Проверяют отдельные функции и методы изолированно
- Используют моки для зависимостей
- Быстро выполняются и точно локализуют проблемы

### Интеграционные тесты
- Проверяют взаимодействие между различными компонентами
- Проверяют интеграцию сервисов с общими утилитами (subprocess_utils, update_utils, permission_utils)
- Проверяют согласованность API между сервисами
- Проверяют интеграцию между интерактивными интерфейсами и сервисами
- Используют метки `@pytest.mark.integration` для идентификации

## Особенности тестов

- Использование фикстур pytest для лучшей организации тестов
- Унифицированные mock-объекты для subprocess результатов
- Изоляция системных зависимостей через mock-объекты
- Повышенная читаемость и поддерживаемость тестов
- Проверка логики обработки системных путей и переменных окружения
- Тесты для утилит командной строки и аргументов запуска
- Использование меток для категоризации тестов (integration/unit/system/slow)
- Совместно используемые фикстуры в conftest.py
- Проверка интерактивных интерфейсов и их взаимодействия с сервисами
