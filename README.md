**Набор Python-скриптов для автоматизации DevOps-задач в Linux-окружении**
Упрощение настройки серверов, оптимизации сетевых параметров и безопасности.

---

## 🚀 Основные возможности

- **Автоматизация настройки сервера**: BBR, Fail2Ban, Docker, TrafficGuard
- **Поддержка дистрибутивов**: Debian, Ubuntu и производные (apt-based)
- **Интерактивный CLI**: удобное меню для выбора операций
- **Идемпотентность**: безопасный многократный запуск
- **Логирование**: подробные логи в файле `app.log`

### 📦 Доступные сервисы

| Сервис | Описание |
|--------|----------|
| **BBR** | Включение/выключение алгоритма перегрузки BBR для оптимизации сети |
| **Docker** | Установка и полное удаление Docker Engine |
| **Fail2Ban** | Защита SSH через настройку Fail2Ban с jail для sshd |
| **TrafficGuard** | Установка и удаление TrafficGuard (защита от сканеров портов) |
| **UV** | Установка и удаление менеджера пакетов UV (Astral) |

## 👥 Пользование (для пользователей)

### Предварительные требования
- Python 3.12+
- Git
- POSIX-совместимая оболочка (bash/zsh)
- **Права root** (требуется `sudo` для выполнения скриптов)

### Установка зависимостей (опционально)
Проект не имеет внешних зависимостей — используются только стандартные модули Python.

### Запуск

#### ⚠️ Требуется root
Все скрипты выполняют системные изменения — запуск только с правами суперпользователя:

```bash
# Через run.sh
sudo ./run.sh

# Или напрямую
sudo python3 main.py

# Или через uv (если установлен)
sudo uv run main.py
```

#### 🚀 Неинтерактивный режим
Вы можете использовать неинтерактивный режим для автоматического выполнения операций без участия пользователя:

```bash
# Установка одного или нескольких сервисов (1=BBR, 2=Docker, 3=Fail2Ban, 4=TrafficGuard, 5=UV)
sudo ./run.sh --install 1 2 3

# Удаление одного или нескольких сервисов
sudo ./run.sh --uninstall 2 3

# Смешанные операции (установка и удаление)
sudo ./run.sh --install 1 5 --uninstall 2

# Режим разработки с подробным логированием
sudo ./run.sh --mode dev --install 1 2
```

#### 🎯 Интерактивное меню
После запуска вам будет предложено выбрать сервис:

## 👨‍💻 Разработка (для разработчиков)

### Предварительные требования
- Docker
- Git

### Работа в DevContainer

Для удобной разработки проекта предусмотрен DevContainer. Это основной и рекомендуемый способ разработки и тестирования скриптов. DevContainer предоставляет изолированную среду разработки с предустановленными инструментами, что исключает необходимость создания отдельных контейнеров для тестирования.

#### Запуск в DevContainer

1. Убедитесь, что у вас установлен Docker
2. Склонируйте репозиторий проекта
3. Запустите сборку контейнера с использованием предоставленного Dockerfile
4. Зайдите в запущенный контейнер для работы с проектом

#### Особенности DevContainer

- Установлен Python 3.12 на базе Debian Bookworm
- Предустановлены все зависимости проекта
- Установлен UV - современный менеджер пакетов Python
- Настроенные инструменты форматирования и линтеры (Black, Flake8)
- Утилиты для разработки (git, common command line utilities)

## 🔧 Режимы работы

```bash
# Продакшен (по умолчанию)
sudo python3 main.py

# Разработка (включает DEBUG-логирование)
sudo python3 main.py --mode dev

# Неинтерактивная установка (установка сервисов 1=BRR, 2=Docker, 3=Fail2Ban и т.д.)
sudo python3 main.py --install 1 2 3

# Неинтерактивное удаление
sudo python3 main.py --uninstall 2 3
```

## 📝 Логирование

Все операции логируются в файл `app.log` с детализацией:
- Время события
- Имя модуля
- Уровень (INFO/DEBUG/ERROR)
- Сообщение

Пример:
```
2025-03-15 10:30:45 - BBRService - INFO - 🚀 Начало включения BBR...
2025-03-15 10:30:46 - BBRService - DEBUG - 📦 Модуль tcp_bbr успешно загружен
2025-03-15 10:30:47 - BBRService - INFO - ✅ BBR успешно включен (алгоритм: bbr) 🎉
```

## 🧪 Тестирование

Проект включает полную систему тестирования:
- **Юнит-тесты**: проверяют отдельные функции и классы изолированно
- **Интеграционные тесты**: проверяют взаимодействие между различными компонентами
- **Системные тесты**: проверяют работу команд и функций в реальной системной среде

### Запуск тестов (внутри Dev Container)
```bash
# Запустить все тесты
pytest -v --cov=app --cov-report=term-missing --tb=short

# Запустить только юнит-тесты
pytest -m unit -v

# Запустить только интеграционные тесты
pytest -m integration -v

# Запустить только системные тесты
pytest -m system -v

# Запустить тесты с подробным логированием (в режиме разработки)
pytest --mode dev -v
```

### Запуск тестов в изолированной среде (для CI/CD)
```bash
docker compose run --rm test pytest -v --cov=app --cov-report=term-missing --tb=short
```
